Mathematics

4. ปูพื้นฐาน: วิธีคูณเมทริกซ์ทางคณิตศาสตร์ (Matrix Multiplication)

การทำนายโมเดลเชิงเส้น ($X_b \theta$) ตั้งอยู่บนฐานของการคูณเมทริกซ์ มาทำความเข้าใจกฎและวิธีคิดด้วยมือก่อนใช้โปรแกรม:

📏 1. กฎตรวจสอบมิติ (Dimension Check)

เราจะคูณเมทริกซ์ $A$ และ $B$ ได้ก็ต่อเมื่อ จำนวนคอลัมน์ของตัวตั้ง เท่ากับ จำนวนแถวของตัวคูณ

$(M \times N) \times (N \times P) = (M \times P)$

ตัวอย่าง: เมทริกซ์ขนาด $100 \times 2$ (ฟีเจอร์บวก Bias) คูณกับเวกเตอร์ขนาด $2 \times 1$ ผลลัพธ์สุดท้ายที่ได้จะมีมิติเท่ากับ $100 \times 1$

🧮 2. วิธีคิดแบบ Dot Product (แถวคูณคอลัมน์)

จับคู่สมาชิกแต่ละตัวของ "แถว (Row)" ในเมทริกซ์แรก คูณกับ "คอลัมน์ (Column)" ในเมทริกซ์หลัง แล้วนำผลลัพธ์มารวมกัน:

$$\begin{bmatrix} a & b \\ c & d \end{bmatrix} \times \begin{bmatrix} x \\ y \end{bmatrix} = \begin{bmatrix} a \cdot x + b \cdot y \\ c \cdot x + d \cdot y \end{bmatrix}$$
ทดลองคำนวณจริง: $$\begin{bmatrix} 1 & 2 \\ 3 & 4 \end{bmatrix} \times \begin{bmatrix} 5 \\ 6 \end{bmatrix} = \begin{bmatrix} 1\cdot5 + 2\cdot6 \\ 3\cdot5 + 4\cdot6 \end{bmatrix} = \begin{bmatrix} 5+12 \\ 15+24 \end{bmatrix} = \begin{bmatrix} 17 \\ 39 \end{bmatrix}$$
NumPy Basics

4.1 การคูณเมทริกซ์และการเขียนคำสั่งแบบเวกเตอร์ใน Python

# สร้างชุดข้อมูลสุ่ม X และ y พร้อมกัน 100 จุดในสเต็ปเดียวโดยไม่ต้องเขียนลูป!
X = 2 * np.random.rand(100, 1)
y = 4 + 3 * X + np.random.randn(100, 1)

📌 คูณเมทริกซ์ใน Python ด้วยเครื่องหมาย @

ตัวดำเนินการ @ ทำการคูณเมทริกซ์แบบแถวคูณคอลัมน์ (Dot Product) ตามทฤษฎีทางคณิตศาสตร์:

# inv(X^T * X) @ X^T @ y (คำนวณ Normal Equation)
theta_best = np.linalg.inv(X_b.T @ X_b) @ X_b.T @ y

*คำเตือน: ห้ามใช้เครื่องหมาย * ในการคูณเมทริกซ์ เพราะจะเป็นการคูณสมาชิกต่อตำแหน่งแบบ Element-wise

📌 ทริกจัดการมิติของ NumPy

  • .T : สลับแถวเป็นคอลัมน์และคอลัมน์เป็นแถว (Transpose)
  • .ravel() : ยุบเมทริกซ์หลายมิติให้เป็นอาร์เรย์ 1 มิติ (Flat)
  • .reshape(row, col) : จัดระเบียบมิติใหม่ เช่น .reshape(-1, 1) จัดเป็นเมทริกซ์แนวตั้ง
NumPy Magic

5. ฟังก์ชันต่อตัวแปรและสร้างตาข่าย (NumPy Magic)

🔗 การต่ออาร์เรย์แบบ np.c_ และ np.r_

  • np.c_[a, b] : นำเวกเตอร์สองก้อนมาประกบเชื่อมกันด้านข้าง (คอลัมน์)
    ในโค้ด: np.c_[np.ones(m), X] เติมคอลัมน์เลข 1 ข้างหน้า X เพื่อทำ Bias
  • np.r_[a, b] : นำเวกเตอร์สองก้อนมาเชื่อมกันด้านบน-ล่าง (แถว)
    ในโค้ด: ใช้ต่อค่าความชันของ Bias กับค่าน้ำหนักโมเดล L2

🌐 np.linspace() & np.meshgrid()

  • np.linspace(-3, 3, 100): สร้างจุดตัวเลข 100 จุดไล่ระดับเฉลี่ยระยะทางคงที่สม่ำเสมอตั้งแต่ -3 ถึง 3 เพื่อสร้างเส้นโค้งในกราฟทำนายผล
  • np.meshgrid(x, y): ผลิตตาข่ายมิติพิกัดแบบ 2 มิติ เพื่อใช้คำนวณจุดทำนายระดับเฉดสีคอนทัวร์แบ่งขอบเขตคลาส

ML Concept

6. สมการเชิงเส้นและทฤษฎี Normal Equation

เป้าหมายคือการหาค่าน้ำหนักพารามิเตอร์ $\hat{\theta}$ ที่ทำให้โมเดลทำนายได้ใกล้เคียงจุดข้อมูลจริงมากที่สุด:

🧮 สมการปิด Normal Equation

$$\hat{\theta} = (X^T X)^{-1} X^T y$$

สูตรนี้หาค่าน้ำหนักพารามิเตอร์ที่ดีที่สุดทางคณิตศาสตร์ได้โดยตรงในสเต็ปเดียว ไม่ต้องทำซ้ำ

ทำไมต้องใช้ Bias Feature ($x_0 = 1$)? เพื่อให้ระบบคูณเมทริกซ์คำนวณค่าจุดตัดแกน Y ($\theta_0$) ได้สะดวกด้วยตัวแปรเดียวกัน

🐍 โค้ด NumPy สำหรับคำนวณสูตรแมนนวล

from sklearn.preprocessing import add_dummy_feature

# 1) เติมคอลัมน์ Bias (x0 = 1) ให้กับแถวข้อมูล
X_b = add_dummy_feature(X)  

# 2) คำนวณตามสูตรทางคณิตศาสตร์:
theta_best = np.linalg.inv(X_b.T @ X_b) @ X_b.T @ y

*คำเตือน: หาก Feature มีจำนวนเยอะมาก เมทริกซ์จะมีขนาดใหญ่ขึ้น และการทำ Inverse จะช้ามากๆ ($O(n^3)$)

Scikit-Learn API

6.1 การรันเชิงเส้นผ่าน Scikit-Learn (วิธีมาตรฐาน)

การเรียนรู้ผ่านระดับสากลด้วยคลาสสำเร็จรูปของ Scikit-Learn สะดวกกว่าและพร้อมนำขึ้นระบบงาน:

🐍 โค้ดภาษา Python

from sklearn.linear_model import LinearRegression

# สร้างโมเดลและป้อนสอนโมเดล (fit)
lin_reg = LinearRegression()
lin_reg.fit(X, y)

# ดูจุดตัดแกน Y และน้ำหนักความชัน
bias = lin_reg.intercept_
coef = lin_reg.coef_

# ทำนายค่าจุดข้อมูลใหม่
predictions = lin_reg.predict(X_new)

📌 อธิบายกลไกเบื้องหลัง

  • fit(X, y) : สั่งสอนและปรับค่าน้ำหนักพารามิเตอร์ให้เข้ากับรูปแบบข้อมูลดิบ
  • predict(X) : นำพารามิเตอร์ที่ได้ไปคำนวณทำนายผลลัพธ์ข้อมูลชุดใหม่
  • เบื้องหลังคลาสนี้จะไม่ได้ใช้ Normal Equation ตรงๆ แต่จะแก้คำตอบด้วยวิธี SVD Least Squares (lstsq) ซึ่งปลอดภัยและทำงานได้มั่นคงกว่าเมื่อพบเมทริกซ์ที่มีความเสี่ยงไม่สามารถ Inverse ได้
Numerical Algebra

6.2 พีชคณิตเชิงเส้นเบื้องหลังโมเดล: SVD และ Pseudoinverse

สคริปต์มีการนำเสนอนิพจน์คำนวณน้ำหนักทางเลือกที่เสถียรกว่าการทำ Inverse แบบปกติ:

🧮 1. ฟังก์ชันกำลังสองน้อยที่สุด (Least Squares SVD)

ใช้ np.linalg.lstsq() เพื่อคำนวณหาน้ำหนัก:

# เบื้องหลังคำนวณหาค่าต่ำสุดของกำลังสองความคลาดเคลื่อน
theta, residuals, rank, s = np.linalg.lstsq(
    X_b, y, rcond=1e-6
)

*ความเสถียร: แก้ไขสมการเชิงเส้นได้แม้คอลัมน์ของข้อมูลจะมีตัวแปรเกี่ยวเนื่องกันแบบเกือบสมบูรณ์ (Singular Matrix)

🛡️ 2. เมทริกซ์ผกผันเทียม Moore-Penrose (Pseudoinverse)

ใช้คำนวณหา Inverse เทียมด้วยตัวดำเนินการ np.linalg.pinv():

# สูตรคำนวณน้ำหนัก: theta = X^+ * y
theta_pinv = np.linalg.pinv(X_b) @ y

*คุณลักษณะ: เป็นการคำนวณตรงๆ ด้วยความเสถียรที่สูงกว่าสมการปิดปกติ เหมาะสำหรับการศึกษาโครงสร้างคณิตศาสตร์เชิงเปรียบเทียบในคลาสเรียน

เปรียบเทียบกระบวนการคำนวณและการอัปเดตน้ำหนักของโมเดลตามความลาดชัน 3 ประเภทหลัก

⬇️ กดปุ่มลูกศรลงเพื่อดูรายละเอียดโค้ดแต่ละประเภท ⬇️

Batch GD Code

7. Batch Gradient Descent: อัปเดตด้วยข้อมูลทั้งหมด

การกวาดหาความชันจากทุกๆ จุดข้อมูลพร้อมกันในแต่ละรอบ (Epoch):

# วนลูปทำการเทรนโมเดลแบบ Batch Gradient Descent
for epoch in range(n_epochs):
    # คำนวณเวกเตอร์ความชัน (Gradients) จากข้อมูลทั้งหมดในคราวเดียว
    gradients = 2 / m * X_b.T @ (X_b @ theta - y)
    # ปรับปรุงพารามิเตอร์ Theta โดยขยับไปในทิศทางตรงกันข้ามกับค่าความชัน
    theta = theta - eta * gradients

ข้อดี: ทางเดินนิ่ง เข้าหาจุดต่ำสุดแน่นอน ข้อเสีย: ล้นหน่วยความจำแรมได้ง่ายมากหากข้อมูลมีจำนวนหลายล้านจุด

Stochastic GD Code

7.1 Stochastic Gradient Descent: ปรับน้ำหนักทีละจุดข้อมูล

สุ่มเลือกข้อมูลทีละแถว และใช้กลไกการปรับลดอัตราการเรียนรู้ให้ค่อยๆ แคบลง (Learning Schedule):

for epoch in range(n_epochs):
    for iteration in range(m):
        # สุ่มหยิบดัชนีของจุดข้อมูลขึ้นมา 1 จุดแบบสุ่ม
        random_index = np.random.randint(m)
        xi = X_b[random_index : random_index + 1]
        yi = y[random_index : random_index + 1]
        # คำนวณค่าความชันเฉพาะจุดข้อมูลนั้นจุดเดียว (ไม่ต้องหารด้วย m ทั้งหมดเหมือน Batch)
        gradients = 2 * xi.T @ (xi @ theta - yi)  
        # คำนวณหาค่า Learning Rate ของสเต็ปปัจจุบันให้ค่อยๆ แคบลง
        eta = learning_schedule(epoch * m + iteration)
        theta = theta - eta * gradients
Mini-batch GD Code

7.2 Mini-batch Gradient Descent: อัปเดตข้อมูลเป็นกลุ่มย่อย

สับเปลี่ยนข้อมูล (Shuffle) ป้องกันการล็อกตำแหน่งและหั่นก้อนย่อยขนาด `minibatch_size`:

for epoch in range(n_epochs):
    # สับเปลี่ยนข้อมูลทั้งหมดแบบสุ่มทุกรอบป้องกันลำดับส่งผลต่อการเรียนรู้
    shuffled_indices = np.random.permutation(m)
    X_b_shuffled = X_b[shuffled_indices]
    y_shuffled = y[shuffled_indices]
    
    for iteration in range(0, n_batches_per_epoch):
        idx = iteration * minibatch_size
        xi = X_b_shuffled[idx : idx + minibatch_size]
        yi = y_shuffled[idx : idx + minibatch_size]
        # คำนวณความชันเฉลี่ยตามขนาดของกลุ่มย่อย
        gradients = 2 / minibatch_size * xi.T @ (xi @ theta - yi)
        eta = learning_schedule(epoch * n_batches_per_epoch + iteration)
        theta = theta - eta * gradients

10. เปรียบเทียบปัญหา: Underfitting vs Overfitting

สองปัญหาพื้นฐานคลาสสิกในการประเมินโมเดลการเรียนรู้เชิงลึก:

📉 1. Underfitting (โมเดลเรียนรู้น้อยเกินไป)

  • คำนิยาม: โมเดลมีความซับซ้อนต่ำเกินไป (High Bias) ไม่สามารถจับแพทเทิร์นที่แท้จริงของข้อมูลได้
  • ผลลัพธ์: มีค่าผิดพลาด (Error) สูงมากทั้งบนข้อมูลที่ใช้สอน (Train) และข้อมูลที่นำมาทดสอบ (Validation/Test)
  • ตัวอย่างในโค้ด: พยายามใช้สมการเส้นตรง (Linear Model) ทายแนวโค้งข้อมูลพาราโบลา
  • แนวทางแก้: เลือกโมเดลที่ซับซ้อนขึ้น, แตกฟีเจอร์พหุนาม

📈 2. Overfitting (โมเดลท่องจำคำตอบ)

  • คำนิยาม: โมเดลมีความซับซ้อนสูงจัด (High Variance) จนเผลอไปจำเอา Noise หรือข้อมูลสุ่มแปรปรวนเข้ามาเป็นกฎหลัก
  • ผลลัพธ์: ทำนายข้อมูลฝึกสอน (Train) ได้เป๊ะเกือบ 100% แต่พอเจอข้อมูลที่ไม่เคยเห็นกลับทำนายผิดพลาดสูงมาก
  • ตัวอย่างในโค้ด: การพล็อตเส้นพหุนามดีกรี 300 วิ่งคดเคี้ยวลากไปโดนจุดสุ่มทุกจุด
  • แนวทางแก้: ปรับลดความซับซ้อนตัวแปรด้วย Regularization, เพิ่มข้อมูล

10.1 รูปภาพจำลองของอาการ Underfitting & Overfitting

เปรียบเทียบลักษณะการทำนายของโมเดลที่ระดับความซับซ้อนต่างกัน (จากรูปในโค้ดทดลอง):

รูปเปรียบเทียบการปรับความซับซ้อน (ดีกรี 1, 2, และ 300)

🔴 ดีกรี 1 (1 Degree) — Underfitting

เส้นตรงสีแดงทื่อ ไม่สามารถโค้งตามจุดข้อมูลจริงได้ โมเดลเรียบง่ายเกินไป (High Bias)

🔵 ดีกรี 2 (2 Degrees) — Good Fit

เส้นประสีน้ำเงินโค้งตามแนวโน้มแท้จริงของข้อมูลพาราโบลาได้พอดี มีความเรียบง่ายและทำนายได้ถูกต้องที่สุด

🟢 ดีกรี 300 (300 Degrees) — Overfitting

เส้นทึบสีเขียวคดเคี้ยวพยายามวิ่งตัดผ่านจุดรบกวน (Noise) ทุกจุด ส่งผลเสียทำให้คำนวณนอกช่วงพลาดสูง (High Variance)

10.2 วิเคราะห์ด้วยกราฟการเรียนรู้จริง (Learning Curves)

เปรียบเทียบลักษณะ RMSE ระหว่างการเทรนจริง (Train) และทดสอบ (Valid) เพื่อวิเคราะห์ความเหมาะสมของโมเดล:

📉 1. Underfitting (High Bias)

เส้นแดง (Train) และสีน้ำเงิน (Valid) วิ่งเข้าหากันอย่างรวดเร็วและนิ่งคงที่ ณ ระดับที่ Error สูงมากทั้งคู่

🎯 2. Good Fit (การเรียนรู้ที่ดี)

เส้นแดง (Train) และสีน้ำเงิน (Valid) ลู่เข้าหากันที่ระดับ Error ต่ำ และไม่มีช่องว่างหรือมีช่องว่างที่แคบมาก

📈 3. Overfitting (High Variance)

เส้นแดง (Train) ลู่ลงระดับเกือบศูนย์ แต่เส้นน้ำเงิน (Valid) ยังสูงและ เกิดช่องว่าง (Gap) กว้างชัดเจน

11. Regularization คืออะไร และทำไปทำไม?

❓ Regularization คืออะไร?

คือการจงใจเพิ่ม "โทษปรับ (Penalty)" เข้าไปในฟังก์ชันประเมินความสูญเสีย (Cost Function) ของโมเดล เมื่อโมเดลพยายามคำนวณค่าน้ำหนักพารามิเตอร์ ($\theta$) ให้มีขนาดใหญ่จนเกินไป

แทนที่โมเดลจะหาจุดที่ฟิตกับข้อมูลฝึกสอน (Training Data) ได้ 100% มันจะถูกบังคับให้เลือกค่าน้ำหนักที่ "เล็กและเรียบง่ายที่สุด" ควบคู่ไปด้วย

🎯 ทำไปเพื่อวัตถุประสงค์อะไร?

  • แก้ไขปัญหา Overfitting (High Variance): ป้องกันไม่ให้โมเดลวาดเส้นคดเคี้ยวไปตาม Noise หรือจุดที่ผิดปกติในข้อมูล
  • ช่วยให้โมเดลประเมินผลได้ทั่วไปขึ้น (Generalization): แม้โมเดลจะมีค่าผิดพลาด (Error) บนข้อมูลฝึกมากขึ้นเล็กน้อย แต่จะสามารถทำนายข้อมูลชุดใหม่ที่ไม่เคยพบมาก่อน (Unseen Data) ได้แม่นยำและมั่นคงขึ้นอย่างมาก
ML Concept

11.1 เจาะลึกสูตรบทลงโทษน้ำหนัก (Ridge, Lasso, Elastic Net)

🛡️ Ridge Regression (L2)

บวกรวมน้ำหนักยกกำลังสอง:

$$\alpha \frac{1}{2} \sum \theta^2$$
  • บีบพารามิเตอร์น้ำหนักให้มีขนาดเล็กลงพร้อมๆ กัน
  • รักษาปัจจัยฟีเจอร์ไว้ครบถ้วน ไม่มีตัวใดโดนหักจนเป็น 0
  • Ridge(alpha=0.1)

🎯 Lasso Regression (L1)

บวกรวมค่าสัมบูรณ์น้ำหนัก:

$$\alpha \sum |\theta|$$
  • บีบพารามิเตอร์และตัดค่าน้ำหนักฟีเจอร์ที่ไม่จำเป็นทิ้งเป็น 0 ทันที
  • คัดเลือกปัจจัยหลักทิ้งปัจจัยรองอัตโนมัติ (Feature Selection)
  • Lasso(alpha=0.1)

🔀 Elastic Net

ผสมผสานรูปแบบ L1 และ L2:

$$r (L1) + (1-r) (L2)$$
  • ควบคุมน้ำหนักได้ยืดหยุ่น
  • เลือกกลุ่มฟีเจอร์ได้สมดุล (Group Selection) เมื่อมีฟีเจอร์ซ้ำซ้อนกันมาก
  • ElasticNet(l1_ratio=0.5)

11.3 ทั้ง 3 รูปแบบควรเลือกทำตอนไหน? (Decision Guide)

แนวทางปฏิบัติในการคัดเลือกสถาปัตยกรรมบทลงโทษให้สอดคล้องกับโจทย์วิศวกรรมข้อมูล:

🛡️ ควรเลือกใช้ Ridge (L2) เมื่อ:

  • เป็นตัวเลือกตั้งต้นหลัก (Default): หากคุณไม่มีความรู้เกี่ยวกับโครงสร้างตัวแปรลึกซึ้ง ให้เริ่มทดลองจาก Ridge ก่อนเสมอ
  • ฟีเจอร์ส่วนใหญ่คาดว่ามีความสำคัญและส่งผลกระทบต่อคำทำนายทีละเล็กละน้อยเหมือนๆ กัน และไม่ต้องการตัดปัจจัยใดออกไปจากการคำนวณ

🎯 ควรเลือกใช้ Lasso (L1) เมื่อ:

  • คุณมีฟีเจอร์จำนวนมาก (เช่น หลายร้อยตัวแปร) แต่เชื่อว่ามี ฟีเจอร์หลักเพียงไม่กี่ตัว เท่านั้นที่ส่งผลจริงต่อผลลัพธ์
  • ต้องการสร้างโมเดลที่มีความบาง (Sparse Model) ทำงานได้รวดเร็ว ประหยัดหน่วยความจำ และตีความวิเคราะห์ผลได้ง่าย

🔀 ควรเลือกใช้ Elastic Net เมื่อ:

  • จำนวนคอลัมน์ฟีเจอร์ มีจำนวนมากกว่าจำนวนแถวข้อมูล ($n > m$) ซึ่งทำให้ Lasso เดี่ยวๆ เริ่มทำงานได้ไม่เสถียร
  • ข้อมูลมีกลุ่มฟีเจอร์ที่มีความสัมพันธ์กันเองสูง (Multicollinearity) เช่น ข้อมูลสรีระนกเพนกวินที่ทั้งน้ำหนัก ขนาดปีก และขนาดปาก มีความแปรผันตามกันหมด Elastic Net จะเลือกทั้งกลุ่มฟีเจอร์ได้ฉลาดกว่า Lasso ที่จะสุ่มหยิบมาเพียงตัวเดียวแล้วตัดตัวอื่นทิ้ง

14. การจำแนกประเภทแบบหลายคลาส (Softmax Regression)

🧮 ฟังก์ชัน Softmax

คำนวณแจกแจงเปอร์เซ็นต์โอกาสในทุกคลาสรวมกันให้เท่ากับ 1.0:

$$\hat{p}_k = \frac{\exp(s_k(x))}{\sum_{j=1}^K \exp(s_j(x))}$$ 
# เขียนแมนนวลเพื่อความปลอดภัยของมิติ:
def softmax(logits):
    exps = np.exp(logits)
    # keepdims=True รักษาแถวไม่ให้มิติอาร์เรย์พัง
    sums = exps.sum(axis=1, keepdims=True)
    return exps / sums

🌸 สไตล์ใช้งานผ่าน Scikit-Learn

โมเดลจะใช้สถาปัตยกรรม Softmax อัตโนมัติเมื่อคลาสในเฉลยมีมากกว่า 2 ตัวแปร:

# ใช้พิกัด 2 มิติทำนาย 3 สายพันธุ์ไอริส
softmax_reg = LogisticRegression(C=30, random_state=42)
softmax_reg.fit(X_train, y_train)

# พยากรณ์ดอกไม้ขนาด [ยาว 5 ซม, กว้าง 2 ซม]

pred_probs = softmax_reg.predict_proba([[5, 2]])

for epoch in range(n_epochs):
    logits = X_train @ Theta          # หาผลคะแนนดิบ
    Y_proba = softmax(logits)         # แปลงเป็นโอกาสน่าจะเป็นด้วยฟังก์ชัน Softmax
    
    # Gradients = X^T * (คำทำนาย - เฉลยจริง)
    error = Y_proba - Y_train_one_hot
    gradients = 1 / m * X_train.T @ error
    
    # ขยับปรับปรุงพารามิเตอร์เมทริกซ์ Theta
    Theta = Theta - eta * gradients

15.2 ระบบบทลงโทษความเข้มข้น L2 & Early Stopping แมนนวล

การนำเอาโทษปรับความกว้างน้ำหนักมารวมใน Gradients และการดักสัญญาณหยุดเรียนรู้ล่วงหน้า:

# 1. บวก Gradients โทษปรับแบบกำลังสอง (ไม่ลงโทษส่วน Bias แถวแรก)
gradients += np.r_[np.zeros([1, n_outputs]), 1 / C * Theta[1:]]

# 2. คำนวณ Loss บน Validation Set และเช็ค Early Stopping
Y_proba_valid = softmax(X_valid @ Theta)
loss = -(Y_valid_one_hot * np.log(Y_proba_valid + epsilon)).sum(axis=1).mean()
total_loss = loss + 1 / (2 * C) * (Theta[1:] ** 2).sum()

if total_loss < best_loss:
    best_loss = total_loss
else:
    # หากรอบปัจจุบันพบจุด Error กลับมาเพิ่มขึ้น ให้สั่งตัดวงจรหยุดสอนงานทันที!
    print(epoch, total_loss, "early stopping!")
    break
