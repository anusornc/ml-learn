ส่วนนี้เพิ่มคอลัมน์ bias `x0 = 1` แล้วใช้สูตรเมทริกซ์เพื่อหา `theta_best` โดยไม่ต้องวนรอบฝึกหลาย epoch

from sklearn.preprocessing import add_dummy_feature

X_b = add_dummy_feature(X)  # add x0 = 1 to each instance
theta_best = np.linalg.inv(X_b.T @ X_b) @ X_b.T @ y

X_new = np.array([[0], [2]])
X_new_b = add_dummy_feature(X_new)
y_predict = X_new_b @ theta_best

plt.plot(X_new, y_predict, "r-", label="Predictions")

Part 4

Scikit-Learn ทำงานแบบ fit แล้ว predict

หลังจากเห็นสูตรด้วย NumPy แล้ว notebook แสดง API ที่ใช้จริงในงาน: สร้าง model, fit ด้วยข้อมูล, แล้ว predict ข้อมูลใหม่

  • `intercept_` คือ bias หรือจุดเริ่มต้นของเส้นเมื่อ feature มีค่า 0
  • `coef_` คือค่าน้ำหนักของ feature
  • `lstsq()` และ `pinv()` เป็นวิธีเชิงเส้นพีชคณิตที่อยู่เบื้องหลัง

bias คืออะไร?

bias คือค่าคงที่ที่โมเดลบวกเพิ่มเข้าไปเสมอ เช่นสมการ `y = 3x + 4` ค่า `4` คือ bias เพราะแม้ `x = 0` โมเดลก็ยังทำนาย `y = 4` ไม่ใช่เริ่มจากศูนย์

`np.linalg.lstsq()` คืออะไร?

ย่อจาก least squares solver: หา `theta` ที่ทำให้ผลรวมกำลังสองของ error ต่ำที่สุด แม้ระบบสมการมีข้อมูลมากกว่าจำนวน unknown เช่น มีจุดข้อมูล 100 จุด แต่มีพารามิเตอร์แค่ 2 ตัว

theta_best_svd: ค่าน้ำหนักที่แก้ได้ เช่น bias และ slope

residuals: ผลรวม error กำลังสองที่เหลือหลังฟิตเส้นแล้ว

rank: จำนวนมิติ/คอลัมน์อิสระจริงของ X_b

s: singular values ใช้บอกความแข็งแรงของแต่ละมิติในข้อมูล

`np.linalg.pinv()` คืออะไร?

คำนวณ pseudoinverse หรือ X+ เพื่อใช้แทน inverse ในกรณีที่เมทริกซ์ไม่เป็น square matrix หรือ inverse ปกติหาไม่ได้ จากนั้นคูณ `X+ @ y` เพื่อได้ค่าน้ำหนักที่ดีที่สุดในเชิง least squares

สรุปสั้น ๆ: `lstsq()` แก้ปัญหา least squares โดยตรง ส่วน `pinv()` สร้างเมทริกซ์ผกผันแบบ generalized ก่อนแล้วค่อยคูณกับ `y`; ทั้งสองช่วยให้โมเดลเสถียรกว่าการใช้ `inv(X.T @ X)` ตรง ๆ เมื่อข้อมูลมี feature ซ้ำซ้อนหรือเมทริกซ์เกือบ singular

from sklearn.linear_model import LinearRegression

lin_reg.fit(X, y)
lin_reg.intercept_, lin_reg.coef_

lin_reg.predict(X_new)

theta_best_svd, residuals, rank, s = np.linalg.lstsq(X_b, y, rcond=1e-6)

Part 5

Gradient Descent: ค่อย ๆ ปรับน้ำหนักให้ error ลดลง

แทนที่จะคำนวณคำตอบตรง ๆ เราเริ่มจาก `theta` สุ่ม แล้วคำนวณ gradient เพื่อบอกว่าควรขยับน้ำหนักไปทิศไหน

eta

learning rate: ก้าวใหญ่หรือเล็ก

n_epochs

จำนวนรอบการเรียนรู้

แกะบรรทัด gradient

Cost function

$$J(\theta)=\frac{1}{m}\sum_{i=1}^{m}\left(h_\theta(x^{(i)})-y^{(i)}\right)^2$$

Cost function คือฟังก์ชันที่ใช้วัดว่าโมเดลผิดมากแค่ไหน ยิ่ง `J(theta)` ต่ำ แปลว่าเส้นทำนายอยู่ใกล้ข้อมูลจริงมากขึ้น

Vector form

$$J(\theta)=\frac{1}{m}\lVert X_b\theta-y\rVert^2$$

Vector form คือการเขียนสูตรเดิมให้คำนวณทุกตัวอย่างพร้อมกันด้วยเมทริกซ์ แทนการวนรวมทีละแถวในเครื่องหมาย Σ

Gradient ที่ต้องหา

$$\nabla_\theta J(\theta)=\frac{2}{m}X_b^T(X_b\theta-y)$$

Gradient ได้จากการหาอนุพันธ์ของ `J(theta)` ตาม `theta` เพื่อบอกทิศทางที่ cost เพิ่มเร็วที่สุด จากนั้นโค้ดใช้ `theta = theta - eta * gradients` เพื่อเดินสวนทางให้ cost ลดลง

  • `X_b @ theta` คือค่าทำนายของโมเดลทุกแถว
  • `X_b @ theta - y` คือ error ระหว่างค่าทำนายกับค่าจริง
  • `X_b.T @ error` รวมผลกระทบของ error กลับไปหาแต่ละพารามิเตอร์
  • `2 / m` มาจากอนุพันธ์ของ Mean Squared Error และการเฉลี่ยตามจำนวนข้อมูล

`@` ใน Python คืออะไร?

`@` คือ matrix multiplication เช่น `A @ B` หมายถึงคูณเมทริกซ์/เวกเตอร์ตามกฎ linear algebra ไม่ใช่คูณสมาชิกต่อสมาชิกแบบ `A * B`

สูตรนี้ไม่ได้เป็น “สูตรของคนเดียว” โดยตรง แต่เกิดจากการหาอนุพันธ์ของ cost function แบบ MSE สำหรับ Linear Regression ส่วนแนวคิด gradient descent เป็นวิธี optimization คลาสสิกที่มักอ้างถึงงานของ Augustin-Louis Cauchy ในศตวรรษที่ 19

eta = 0.1
n_epochs = 1000
m = len(X_b)

np.random.seed(42)
theta = np.random.randn(2, 1)

for epoch in range(n_epochs):
    gradients = 2 / m * X_b.T @ (X_b @ theta - y)
    theta = theta - eta * gradients

Part 6

SGD และ Mini-batch ลดต้นทุนต่อรอบ

Batch ใช้ข้อมูลทั้งหมดทุกครั้ง ส่วน SGD ใช้ทีละตัวอย่าง และ Mini-batch ใช้ทีละกลุ่มเล็ก ๆ จึงเร็วกว่าเมื่อข้อมูลใหญ่

Batch GD

เลือกเมื่อข้อมูลไม่ใหญ่มากและต้องการ update ที่นิ่ง เพราะคำนวณ gradient จากข้อมูลทั้งหมด แต่หนึ่งรอบจะช้าถ้าข้อมูลเยอะ

SGD

เลือกเมื่อข้อมูลใหญ่มากหรือต้องการเริ่มเรียนรู้เร็ว ใช้ทีละตัวอย่างจึงเร็วต่อ update แต่เส้นทางการเรียนรู้จะแกว่งกว่า

Mini-batch GD

มักเป็นตัวเลือกหลักในงานจริง เพราะสมดุลระหว่างความเร็วและความนิ่ง ใช้เวกเตอร์/GPU ได้ดี และไม่แกว่งเท่า SGD

`learning_schedule()` ทำให้ learning rate ค่อย ๆ ลดลง เพื่อลดการกระโดดรอบจุดเหมาะสม

# Batch GD: ใช้ข้อมูลทั้งหมดในหนึ่ง update
for epoch in range(n_epochs):
    gradients = 2 / m * X_b.T @ (X_b @ theta - y)
    theta = theta - eta * gradients

# SGD: สุ่มข้อมูลทีละ 1 แถวในหนึ่ง update
for epoch in range(n_epochs):
    for iteration in range(m):
        random_index = np.random.randint(m)
        xi = X_b[random_index : random_index + 1]
        yi = y[random_index : random_index + 1]
        gradients = 2 * xi.T @ (xi @ theta - yi)
        eta = learning_schedule(epoch * m + iteration)
        theta = theta - eta * gradients

# Mini-batch GD: ใช้ข้อมูลทีละกลุ่มเล็ก ๆ
for epoch in range(n_epochs):
    shuffled_indices = np.random.permutation(m)
    X_b_shuffled = X_b[shuffled_indices]
    y_shuffled = y[shuffled_indices]
    for iteration in range(0, n_batches_per_epoch):
        idx = iteration * minibatch_size
        xi = X_b_shuffled[idx : idx + minibatch_size]
        yi = y_shuffled[idx : idx + minibatch_size]
        gradients = 2 / minibatch_size * xi.T @ (xi @ theta - yi)
        theta = theta - eta * gradients

Part 7

Polynomial Regression ทำให้เส้นตรงจับรูปโค้งได้

แนวคิดคือสร้าง feature ใหม่ เช่น `x^2` แล้วให้ Linear Regression เรียนรู้บน feature ที่ขยายแล้ว

โมเดลยังเป็น linear ในเชิงพารามิเตอร์ แต่เส้นที่วาดบนแกน x กลายเป็นเส้นโค้ง

ทำไมต้องใช้ Polynomial?

ถ้าจุดข้อมูลมีรูปแบบโค้ง เส้นตรงธรรมดาจะ underfit เพราะอธิบายความสัมพันธ์ได้แค่แนวเพิ่ม/ลดเป็นเส้นตรง วิธีแก้คือเพิ่ม feature ที่บอกความโค้ง เช่น `x²`, `x³` แล้วให้ Linear Regression หา coefficient ของ feature เหล่านี้

พูดง่าย ๆ: เราไม่ได้เปลี่ยน algorithm หลัก แต่เปลี่ยน “หน้าตาข้อมูล input” ให้โมเดลมีข้อมูลพอจะวาดเส้นโค้งได้

ภาพเปรียบเทียบเส้นตรงกับเส้นโค้ง

Linear Polynomial 
from sklearn.preprocessing import PolynomialFeatures

poly_features = PolynomialFeatures(degree=2, include_bias=False)
X_poly = poly_features.fit_transform(X)

lin_reg = LinearRegression()
lin_reg.fit(X_poly, y)

X_new = np.linspace(-3, 3, 100).reshape(100, 1)
X_new_poly = poly_features.transform(X_new)
y_new = lin_reg.predict(X_new_poly)

ทำไมต้องใช้ `np.linspace(-3, 3, 100).reshape(100, 1)`?

`np.linspace(-3, 3, 100)` สร้างค่า x ใหม่ 100 จุด ตั้งแต่ -3 ถึง 3 เพื่อเอาไปวาดเส้น prediction ให้ต่อเนื่องและเรียบ ไม่ใช่ทำนายแค่จุดข้อมูลเดิม

`.reshape(100, 1)` เปลี่ยน array จากรูปแบบแถวเดียว `(100,)` ให้เป็น column vector `(100, 1)` เพราะ Scikit-Learn คาดว่า input ต้องมีรูป `(จำนวนตัวอย่าง, จำนวน feature)`

Part 8

Learning Curves ใช้ดู underfitting และ overfitting

โค้ดวัด RMSE ของ train และ validation เมื่อเพิ่มจำนวนข้อมูลฝึก ถ้าเส้นทั้งสองสูงใกล้กันคือ underfitting ถ้าห่างกันมากมักเป็น overfitting

ส่วนนี้สอนให้นักศึกษาไม่ดูแค่คะแนน train แต่ต้องดู validation ด้วย

RMSE คืออะไร?

$$RMSE=\sqrt{\frac{1}{m}\sum_{i=1}^{m}\left(\hat{y}^{(i)}-y^{(i)}\right)^2}$$

RMSE คือรากที่สองของค่า error กำลังสองเฉลี่ย หน่วยจะกลับมาเป็นหน่วยเดียวกับ target เช่น ถ้าทำนายราคา RMSE ก็อยู่ในหน่วยราคา

from sklearn.model_selection import learning_curve

train_sizes, train_scores, valid_scores = learning_curve(
    LinearRegression(), X, y,
    train_sizes=np.linspace(0.01, 1.0, 40),
    cv=5,
    scoring="neg_root_mean_squared_error")  # Scikit-Learn ใช้คะแนนติดลบ

# กลับเครื่องหมายลบให้เป็นค่า RMSE จริง เพราะ error ยิ่งน้อยยิ่งดี
train_errors = -train_scores.mean(axis=1)
valid_errors = -valid_scores.mean(axis=1)

หมายเหตุ: `neg_root_mean_squared_error` คือ RMSE ที่ใส่เครื่องหมายลบ เพราะระบบ scoring ของ Scikit-Learn มองว่า “คะแนนยิ่งมากยิ่งดี” แต่ RMSE เป็น error ที่ “ยิ่งน้อยยิ่งดี” จึงต้องใส่ `-` หน้า score เพื่อแปลงกลับเป็น RMSE จริง

ใช้ RMSE เมื่อ

อยากอ่าน error ในหน่วยเดียวกับ target และต้องการลงโทษ error ใหญ่ ๆ มากเป็นพิเศษ

ใช้ MSE เมื่อ

ต้องการ cost function ที่ smooth สำหรับ optimization แต่ค่าที่ได้เป็นหน่วยยกกำลังสองจึงตีความยากกว่า

ใช้ MAE เมื่อ

ข้อมูลมี outlier เยอะ และไม่อยากให้ error ใหญ่บางจุดครอบงำผลรวมมากเกินไป

Metric อื่น ๆ

ถ้าเป็น classification ให้ใช้ accuracy, precision, recall, F1 หรือ ROC-AUC ตามเป้าหมายของปัญหา

Part 9

Regularization คือการลงโทษโมเดลที่ซับซ้อนเกินไป

Ridge ใช้ L2 penalty, Lasso ใช้ L1 penalty และ Elastic Net ผสมทั้งสองแบบ เพื่อช่วยลด overfitting โดยเฉพาะเมื่อใช้ polynomial degree สูง

  • Ridge ลดน้ำหนักให้เล็กลง
  • Lasso ดันบางน้ำหนักเป็นศูนย์ จึงช่วยเลือก feature
  • Elastic Net ประนีประนอมระหว่าง Ridge และ Lasso

ชื่อแปลก แต่ idea ง่าย

ทั้งสามตัวคือ Linear Regression ที่เพิ่ม “ค่าปรับ” เข้าไปใน cost function เพื่อบอกโมเดลว่าอย่าใช้ coefficient ใหญ่เกินจำเป็น เพราะ coefficient ใหญ่ทำให้เส้นแกว่งและ overfit ได้ง่าย

เกร็ดชื่อเรียก

Ridge มาจาก ridge regression ที่เพิ่มค่าบนแนวทแยงของเมทริกซ์ ทำให้การแก้สมการนิ่งขึ้น คล้ายสร้าง “สัน” ช่วยพยุงปัญหาที่ไม่เสถียร

Lasso ย่อจาก Least Absolute Shrinkage and Selection Operator คำว่า lasso สื่อถึงบ่วงที่รัด coefficient ให้เล็กลง และบางตัวถูกดึงเหลือศูนย์

Elastic Net สื่อถึงตาข่ายยืดหยุ่นที่ผสมแรงรัดแบบ Lasso กับแรงดึงให้นุ่มแบบ Ridge

ต้องใช้ทั้งสามตัวไหม?

ไม่ต้องใช้พร้อมกัน ปกติเลือกตัวใดตัวหนึ่งแล้วทดลองปรับ `alpha` ด้วย validation set หรือ cross-validation

เริ่มจาก Ridge ถ้ายังอยากเก็บทุก feature, ใช้ Lasso ถ้าอยากให้โมเดลช่วยตัด feature, และใช้ Elastic Net เมื่อ feature เยอะหรือสัมพันธ์กันสูง

from sklearn.linear_model import Ridge, Lasso, ElasticNet

ridge_reg = Ridge(alpha=0.1, solver="cholesky")
ridge_reg.fit(X, y)

lasso_reg = Lasso(alpha=0.1)
lasso_reg.fit(X, y)

elastic_net = ElasticNet(alpha=0.1, l1_ratio=0.5)
elastic_net.fit(X, y)

Ridge = L2 penalty

$$J(\theta)+\alpha\sum_j\theta_j^2$$

ลงโทษน้ำหนักใหญ่ด้วยกำลังสอง ทำให้น้ำหนักเล็กลงแบบนุ่ม ๆ แต่โดยมากไม่เป็นศูนย์

Lasso = L1 penalty

$$J(\theta)+\alpha\sum_j|\theta_j|$$

ลงโทษด้วยค่าสัมบูรณ์ จึงมักดัน feature ที่ไม่สำคัญให้ coefficient เป็น 0 ได้

Elastic Net = L1 + L2

ผสมข้อดีของ Ridge และ Lasso ใช้ `l1_ratio` บอกว่าจะเอียงไปทาง Lasso มากแค่ไหน และ `alpha` บอกแรงลงโทษรวม

วิธีใช้จริง: แทน `LinearRegression()` ด้วย `Ridge`, `Lasso` หรือ `ElasticNet` แล้วปรับ `alpha`; ค่า `alpha` มากขึ้นทำให้โมเดลเรียบขึ้นแต่อาจ underfit ส่วน `l1_ratio` ใช้เฉพาะ Elastic Net โดยใกล้ 1 จะคล้าย Lasso และใกล้ 0 จะคล้าย Ridge

def softmax(logits):
    exps = np.exp(logits)
    exp_sums = exps.sum(axis=1, keepdims=True)
    return exps / exp_sums

for epoch in range(n_epochs):

Part 15

Step 2: หา slope หรือความชันของเส้น

slope คือคะแนนที่คาดว่าจะเพิ่มขึ้นเมื่อชั่วโมงเรียนเพิ่ม 1 ชั่วโมง สูตรนี้คือสูตร OLS สำหรับเส้นตรงหนึ่งตัวแปร

สูตรหา slope

$$b_1=\frac{n\sum xy-\sum x\sum y}{n\sum x^2-(\sum x)^2}$$

แทนค่าจากตาราง

$$b_1=\frac{9(3065)-45(510)}{9(285)-45^2}$$ $$b_1=\frac{27585-22950}{2565-2025}=\frac{4635}{540}=8.5833$$

ดังนั้น slope = 8.58

แปลว่า ถ้านักศึกษาเรียนเพิ่ม 1 ชั่วโมง โมเดลจะทำนายว่าคะแนนเพิ่มประมาณ 8.58 คะแนน โดยเฉลี่ยตามข้อมูลชุดนี้

Excel:
=((9*3065)-(45*510))/((9*285)-(45^2))

Part 16

Step 3: หา intercept หรือจุดเริ่มต้นของเส้น

เมื่อได้ slope แล้ว intercept คือค่าคงที่ที่ทำให้เส้นวางอยู่ในตำแหน่งที่เหมาะกับข้อมูล ไม่จำเป็นต้องผ่านจุดศูนย์

สูตรหา intercept

$$b_0=\frac{\sum y-b_1\sum x}{n}$$

หรือจำง่าย ๆ ว่า `intercept = ค่าเฉลี่ย y - slope * ค่าเฉลี่ย x`

แทนค่า slope ที่หาได้

$$b_0=\frac{510-(8.5833)(45)}{9}$$ $$b_0=\frac{510-386.25}{9}=\frac{123.75}{9}=13.75$$

ดังนั้น intercept = 13.75

แปลว่า ถ้า `x = 0` โมเดลจะเริ่มทำนายที่ 13.75 คะแนน ซึ่งเป็นค่าตั้งต้นของเส้น ไม่ใช่คะแนนจริงที่ต้องเกิดขึ้นเสมอ

Excel:
=(510-(8.583333*45))/9
